  2. The biggest advantage the the D3 library provided was the ability to make dynamic graphs, which allows for the compact visualization of multiple graphs in a clean and efficient way. Rather than having to display 6 graphs for sales broken down by genre (one for each region), one is able to see a single graph that shifts based on the specific region of interest.

    The second advtange of using D3 is the sheer custimizability of the library. By providing a vast number of graph frameworks to use, the designer has lots of choices on how best they want to display their data. For instance, I was able to customize the color of the bars in my bar charts very easily, highlighting bars which corresponded to different values on my graph. Other libraries (such as matplotlib in python) do not allow for such freedom, whereas D3 provides a plethora of options for users.

    The third advantage of using D3 was the ability to integrate my graphs directly into the HTML page. This allows for viewers to interact with the data, adding an "exploratory" aspect to the data which other libraries fail to provide. This allowed me to create more interesting graphs which draw the user in, rather than creating stale, static graphs which fail to hold people's attention.

  3. The biggest disadvantage of using D3 is how inefficient the libary is, especially in respect to time. Because each graph can require unique customization, creating a large number of graphs for data visualization can be unnecessarily complicated and time consuming. The coder must have have complete mastery of the D3 library (which has a steep learning curve) and even then, coding each specific part of a graph can take a fair amount of time, given how much customization is possible.

    The second disadvantage of using D3 is that the updates to the graph take place in real time, meaning the thousands of calculations must be done per second to update the graph. If the dataset being used in a graph is very large (on the scale of millions), updating the graphs may take a very large amount of time top update, causing lag and other delays.

    The third disadvantage of using D3 (specifically the dynamic graphs) is that it makes side-by-side comparisons of data much harder to visualize. For instance, if the user desires to see what sales by genre in Japan and Europe like, such comparisons can not be done directly. A user must first look at sales in Japan, then Europe, and there is no easy way of view such data at the same time. While these graphs theoretically could have been programmed to be side by side, the amount of time and code this would takes makes the whole process inefficient and overall a waste of time and space.

  4. My dashboard is relatively accessible to most users, displaying data in a clean, efficient way. The color schemes are distinct and complement eachother nicely, while not being too distracting.
